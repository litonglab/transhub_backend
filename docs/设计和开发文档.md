# Transhub 后端系统设计和开发文档

## 1. 技术栈

### 1.1 核心框架与库

- **Web框架**: Flask 3.1.1
- **数据库ORM**: SQLAlchemy 2.0.41 + Flask-SQLAlchemy 3.1.1
- **数据库驱动**: PyMySQL 1.1.1 (MySQL)
- **任务队列**: Dramatiq 1.18.0 + dramatiq_dashboard 0.4.0
- **缓存**: Redis 4.6.0 + Flask-Redis 0.4.0 + Flask-Caching 2.3.1
- **认证**: Flask-JWT-Extended 4.7.1 (JWT Token存储在Cookie中)
- **参数校验**: Pydantic 2.11.5
- **跨域支持**: Flask-CORS 6.0.1
- **进程管理**: Supervisor 4.2.5
- **WSGI服务器**: Gunicorn 23.0.0
- **日志**: concurrent-log-handler 0.9.28 (支持多进程日志)
- **环境变量**: python-dotenv 1.1.0
- **数据可视化**: Matplotlib 3.5.3, CairoSVG 2.8.2
- **系统监控**: psutil 7.0.0

### 1.2 技术特点

- **单例模式**: Flask应用采用线程安全的单例模式，支持多进程部署
- **异步任务**: 使用Dramatiq实现异步任务队列，支持任务重试和超时控制
- **分布式锁**: 基于Redis实现分布式锁，保证并发安全
- **缓存策略**: 多层缓存设计，包括用户认证缓存、榜单缓存等
- **日志系统**: 双重日志机制（文件+控制台），支持日志轮转和多进程写入

## 4. Redis与缓存

### 4.1 Redis配置

- **连接库**: Flask-Redis 0.4.0
- **缓存库**: Flask-Caching 2.3.1
- **连接方式**: 通过环境变量配置Redis URL
- **超时设置**: 连接超时5秒，支持超时重试

### 4.2 缓存应用场景

#### 4.2.1 用户认证缓存

```python
@cache.memoize(timeout=10 * 60)  # 缓存10分钟
def find_by_id_for_auth(user_id):
    """用户认证信息缓存，减少数据库查询"""
    return UserModel.query.get(user_id)
```

**特点**:

- JWT认证时自动从缓存加载用户信息
- 用户信息更新时自动清除缓存（`reset_user_auth_cache()`）
- 缓存时间：10分钟

#### 4.2.2 榜单缓存

```python
@cache.cached(timeout=10 * 60, key_prefix='rank_list')
def get_rank_list(cname):
    """榜单数据缓存，减少数据库查询"""
    return RankModel.query.filter_by(cname=cname).all()
```

**特点**:

- 榜单更新时自动清除缓存（`reset_rank_cache()`）
- 缓存时间：10分钟
- 支持按课程名称分别缓存

#### 4.2.3 分布式锁

```python
# 端口分配锁
lock_name = f'port_lock_{port}'
lock = Lock(redis_client, lock_name, timeout=600)
with lock:
    # 分配端口逻辑

# 编译锁（课程级别）
lock_name = f'compile_lock_{cname}'
lock = Lock(redis_client, lock_name, timeout=300)
with lock:
    # 编译代码逻辑

# 榜单更新锁（用户级别）
lock_name = f'rank_update_lock_{user_id}_{cname}'
lock = Lock(redis_client, lock_name, timeout=30)
with lock:
    # 更新榜单逻辑
```

**特点**:

- 端口分配锁：防止端口冲突，超时600秒
- 编译锁：防止同一课程多个任务同时编译，超时300秒
- 榜单更新锁：防止同一用户并发更新榜单，超时30秒

### 4.3 性能优化

#### 4.3.1 缓存策略

- **用户认证**: 高频访问，缓存10分钟
- **榜单数据**: 中频访问，缓存10分钟
- **配置信息**: 低频变更，应用启动时加载

#### 4.3.2 缓存失效

- **主动失效**: 数据更新时主动清除相关缓存
- **被动失效**: 设置合理的过期时间，自动失效
- **缓存预热**: 应用启动时预加载热点数据

## 5. 任务队列

### 5.1 Dramatiq配置

- **消息代理**: Redis
- **任务队列**: 三个独立队列
  - `cc_training`: 拥塞控制算法评测任务
  - `graph`: 图表生成任务
  - `svg2png`: SVG转PNG任务
- **超时控制**: 20分钟（1200000毫秒）
- **重试策略**: 不重试（`max_retries=0`）

### 5.2 任务队列枚举

```python
class DramatiqQueue(Enum):
    CC_TRAINING = "cc_training"  # 运行算法评测的任务队列
    GRAPH = "graph"              # 绘图任务队列
    SVG2PNG = "svg2png"          # 时延图svg转png的任务队列
```

### 5.3 核心任务：拥塞控制算法评测

#### 5.3.1 任务流程

```
1. 检查任务状态（QUEUED）
2. 编译代码（COMPILING → COMPILED）
   - 获取编译锁（课程级别）
   - 检查是否已编译
   - 执行make编译
   - 移动二进制文件到任务目录
3. 运行评测（RUNNING）
   - 分配可用端口
   - 执行run-contest.sh脚本
   - 记录运行日志
4. 评分（evaluate_score）
   - 解析日志文件
   - 计算丢包率、时延、吞吐量
   - 计算总分
5. 更新榜单（FINISHED）
   - 获取榜单更新锁（用户级别）
   - 检查所有任务是否完成
   - 更新或创建榜单记录
   - 清除榜单缓存
6. 生成图表（异步）
   - 将图表任务放入graph队列
```

#### 5.3.2 任务状态机

```
NOT_QUEUED → QUEUED → COMPILING → COMPILED → RUNNING → FINISHED
                ↓          ↓          ↓          ↓
              ERROR    COMPILED_FAILED  ERROR    ERROR
```

**状态优先级**（用于历史记录聚合）:

```
COMPILED_FAILED > ERROR > COMPILED > COMPILING > RUNNING > QUEUED > NOT_QUEUED > FINISHED
```

#### 5.3.3 异常处理

```python
try:
    # 任务执行逻辑
except TimeLimitExceeded:
    # Dramatiq超时异常（20分钟）
    _handle_exception(task_id, err_msg, ...)
except Exception as e:
    # 其他异常
    _handle_exception(task_id, err_msg, ...)
finally:
    # 清理资源
    db.session.remove()
    release_port(running_port, redis_client)
```

### 5.4 锁机制

#### 5.4.1 端口分配锁

```python
lock_name = f'port_lock_{port}'
result = redis_client.set(lock_name, "locked", nx=True, ex=600)
```

**特点**:

- 端口范围：50000-65535
- 锁超时：600秒（单个任务运行时间）
- 自动释放：任务完成后释放

#### 5.4.2 编译锁

```python
lock_name = f'compile_lock_{cname}'
lock = Lock(redis_client, lock_name, timeout=300)
with lock:
    # 编译逻辑
```

**特点**:

- 课程级别锁：同一课程同时只能有一个任务编译
- 锁超时：300秒
- 编译优化：检查是否已编译，避免重复编译

#### 5.4.3 用户目录锁

```python
lock_name = f'task_dir_lock_{upload_id}'
user_lock = Lock(redis_client, lock_name, timeout=300)
with user_lock:
    # 检查已编译文件
```

**特点**:

- 用户级别锁：防止同一用户多个任务同时读写二进制文件
- 锁超时：300秒
- 避免等待：已编译的文件可直接使用

#### 5.4.4 榜单更新锁

```python
lock_name = f'rank_update_lock_{user_id}_{cname}'
rank_lock = Lock(redis_client, lock_name, timeout=30)

```

**特点**:

- 用户+课程级别锁：防止同一用户并发更新榜单
- 锁超时：30秒
- 更新策略：只有所有任务完成且分数更高时才更新

## 6. 接口参数校验

### 6.1 校验框架

- **校验库**: Pydantic 2.11.5
- **装饰器**: `@validate_request(SchemaClass)`
- **支持类型**: GET参数、POST JSON、表单数据、文件上传

### 6.2 校验装饰器

```python
@validate_request(UserLoginSchema)
def login():
    data = request.validated_data  # 类型为 UserLoginSchema
    username = data.username
    password = data.password
```

**特点**:

- 自动识别请求类型（GET/POST/文件上传）
- 校验失败返回详细错误信息
- 支持类型提示，IDE自动补全

### 6.3 校验模型示例

#### 6.3.1 用户登录校验

```python
class UserLoginSchema(BaseModel):
    username: str = Field(..., min_length=4, max_length=50)
    password: str = Field(..., min_length=6, max_length=18)
    cname: str = Field(..., description="比赛名称")

    @field_validator('username')
    def validate_username(cls, v):
        # 检查用户名格式（中文、字母、数字、下划线、点号）
        # 不能以点号开头，不能包含连续点号
        # 不能包含文件系统特殊字符
        return CommonValidators.validate_username(v)

    @field_validator('cname')
    def validate_cname(cls, v):
        if v not in config.Course.CNAME_LIST:
            raise ValueError(f'比赛名称必须是以下之一: {", ".join(config.Course.CNAME_LIST)}')
        return v
```

#### 6.3.2 文件上传校验

```python
class FileUploadSchema(BaseModel):
    file: Optional[object] = Field(None, description="上传的文件")
    trace_list: str = Field(..., description="Trace列表(JSON字符串)")

    @field_validator('file')
    def validate_file(cls, file):
        # 1. 检查文件是否存在
        # 2. 检查文件后缀（.c, .cc, .cpp）
        # 3. 检查文件名格式（中文、字母、数字、下划线、点号、连字符）
        # 4. 检查文件大小（最大2MB）
        # 5. 检查文件内容安全性（危险函数检测）
        return file

    @classmethod
    def _validate_file_content_safety(cls, file):
        """检查文件是否包含危险函数调用"""
        dangerous_functions = [
            "fopen", "system", "execve", "socket", "dlopen", ...
        ]
        for func in dangerous_functions:
            if re.search(rf'\b{re.escape(func)}\s*\(', code):
                raise ValueError(f"文件包含危险函数调用: {func}")
```

def evaluate_score(task, result_path):
    """评估任务分数"""
    # 1. 解析日志文件
    metrics = parse_log_file(result_path)

    # 2. 计算各维度分数
    loss_score = calculate_loss_score(metrics['loss_rate'])
    delay_score = calculate_delay_score(metrics['avg_delay'])
    throughput_score = calculate_throughput_score(metrics['throughput'])

    # 3. 加权求和
    weights = config.get_course_trace_config(task.cname, task.trace_name)['score_weights']
    total_score = (
        loss_score * weights['loss'] +
        delay_score * weights['delay'] +
        throughput_score * weights['throughput']

    # 4. 更新任务分数
    task.update(
        task_score=total_score,
        loss_score=loss_score,
        delay_score=delay_score,
        throughput_score=throughput_score
    )

### 11.6 榜单更新策略

```python
def _update_rank(task, user):
    """更新榜单"""
    # 1. 获取榜单更新锁（用户级别）
    lock_name = f'rank_update_lock_{user_id}_{cname}'

        if not all(t.task_status == TaskStatus.FINISHED for t in all_tasks):

        total_score = sum(t.task_score for t in all_tasks)

        rank_record = RankModel.query.filter_by(competition_id=task.competition_id).first()
        if rank_record:
            # 只有分数更高时才更新

                rank_record.update(
                    upload_id=upload_id,
                    task_score=total_score,
                    algorithm=task.algorithm,
                    upload_time=task.created_time
                )
        else:
            # 创建新记录
            RankModel(...).insert()

        # 5. 清除榜单缓存
        reset_rank_cache(task.cname)
```

**特点**:

- 只有所有任务完成后才更新榜单
- 只保留最高分记录
- 使用分布式锁保证并发安全
